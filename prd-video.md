> 구현 스펙(DB 스키마, API, 프로젝트 구조 등)은 [spec.md](./spec.md) 참고

---

## 1. 개요

어드민이 동영상을 업로드하고, 재생 목록을 구성하여 시청자가 유튜브처럼 자유롭게 시청하는 VOD 서비스.

- 어드민이 동영상을 업로드하면 자동 인코딩 후 시청 가능
- 재생목록을 만들어 영상을 순서대로 재생
- 시청자는 원하는 영상을 골라 보거나, 재생목록을 순서대로 시청
- 각 시청자는 독립적으로 재생 (동기화 없음)

### 운영 제약

| 항목 | 값 |
|------|-----|
| 최대 동시 시청자 | **500명** |
| 기본 영상 화질 | **1080p** (5Mbps) |
| 인프라 비용 목표 | **최소 비용 운영** |

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| Backend | Spring Boot 4.0.5, Java 25, JPA, MySQL, Lombok |
| Frontend | Next.js (App Router), React, TypeScript, Tailwind CSS |
| 서버 | NCP Server (VM) 1대 |
| 저장소 | 서버 로컬 디스크 (HLS 서빙) + NCP Object Storage (원본 백업) |
| Build | Gradle (Kotlin DSL) |
| 인증 | JWT (Access Token + Refresh Token) |
| 영상 재생 | HLS.js |
| 영상 인코딩 | FFmpeg (서버 인코딩, 무료) |
| 웹 서버 | Nginx (HLS 정적 파일 서빙) |

## 3. 인프라 구성

### 3.1 단일 서버 아키텍처

```
┌─────────────────────────────────────────────────────┐
│              NCP Server (VM) 1대                     │
│                                                      │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────┐  │
│  │ Nginx       │  │ Spring Boot  │  │ MySQL      │  │
│  │ (HLS 서빙)  │  │ (API)       │  │ (Docker)   │  │
│  │ :80/:443    │  │ :8080        │  │ :3306      │  │
│  └──────┬──────┘  └──────┬───────┘  └────────────┘  │
│         │                │                           │
│  ┌──────┴────────────────┴───────┐                   │
│  │      로컬 디스크 / Block Storage  │                │
│  │  /data/videos/encoded/  (HLS)  │                  │
│  │  /data/videos/originals/ (원본) │                  │
│  │  /data/videos/thumbnails/      │                  │
│  └────────────────────────────────┘                  │
│                                                      │
│  FFmpeg (업로드 시 비동기 인코딩)                       │
└──────────────────────┬───────────────────────────────┘
                       │
              인터넷 (1Gbps+)
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
       시청자 A     시청자 B     시청자 C
        (각자 독립 재생, 동기화 없음)
```

### 3.2 NCP Object Storage (백업용만)

- 원본 영상 파일 백업 저장 (서버 장애 대비)
- 서빙 용도로는 사용하지 않음 → **전송 비용 0원**
- 저장 비용만 발생: ~23원/GB/월

### 3.3 서버 사양 권장

| 항목 | 최소 사양 | 권장 사양 |
|------|----------|----------|
| CPU | 4 vCPU | 8 vCPU (인코딩 시 CPU 사용) |
| RAM | 8GB | 16GB |
| 디스크 | 500GB SSD | 1TB SSD (영상 파일 저장) |
| 네트워크 | 1Gbps | 2Gbps (여유분) |

## 4. 핵심 동작 흐름

### 4.1 영상 업로드 ~ 시청

```
[어드민] 동영상 업로드
    │ multipart upload
    ▼
[Backend]
  1. 원본 파일 로컬 저장 (/data/videos/originals/)
  2. Video 엔티티 생성 (status: UPLOADING → ENCODING_QUEUED)
  3. (비동기) Object Storage에 원본 백업
  4. FFmpeg 비동기 인코딩 실행
     → 1080p HLS 세그먼트 생성 (.m3u8 + .ts)
     → /data/videos/encoded/{videoId}/
     → 썸네일 자동 추출
  5. Video 엔티티 갱신 (status: READY, hlsPath 설정)
    │
    ▼
[시청자]
  1. 영상 목록/재생목록에서 영상 선택
  2. Backend에서 HLS URL + 메타데이터 응답
  3. HLS.js가 Nginx에서 HLS 세그먼트 로드
  4. 자유롭게 재생 (탐색, 일시정지 등)
```

### 4.2 재생목록 재생

```
[시청자] 재생목록 선택
    │
    ▼
[Frontend]
  1. 재생목록의 영상 목록 조회 (순서 포함)
  2. 첫 번째 영상부터 HLS.js로 재생
  3. 영상 종료 시 → 자동으로 다음 영상 재생 (프론트엔드 제어)
  4. 시청자가 목록에서 원하는 영상 클릭 → 해당 영상으로 이동

  * 재생목록 내 이전/다음 영상 이동은 프론트엔드에서 처리
  * 서버는 재생목록과 영상 순서 정보만 제공
```

## 6. 비용 추정

### 6.1 월간 비용

| 항목 | 비용 |
|------|------|
| NCP Server (8vCPU, 16GB, 1TB SSD) | ~20~40만원/월 |
| NCP 서버 아웃바운드 전송 | 서버 요금제에 따라 다름 |
| NCP Object Storage 저장 (원본 백업 500GB) | ~1.2만원/월 |
| Object Storage 전송 | **0원** (서빙 안 함) |
| 도메인 + SSL | ~1~2만원/월 |

### 6.2 대역폭 참고

```
P2P 없이 최악의 경우:
  500명 × 5Mbps = 2.5Gbps (서버 감당 불가)

실제 VOD 패턴:
  - 모든 시청자가 동시에 같은 영상을 보지 않음
  - 피크 시간에도 동시 재생 200~300명 수준으로 예상
  - 200명 × 5Mbps = 1Gbps → 1Gbps 서버로 감당 가능

만약 대역폭 부족 시:
  - Nginx 캐시 최적화
  - Cloudflare 무료 플랜 (대역폭 무제한) 프록시 검토
  - NCP CDN+ 도입 검토
```

## 7. 주의사항 및 제약

- **서버 대역폭**: VOD는 시청자별 재생 시점이 달라 피크가 분산됨. 그래도 1Gbps 서버 권장
- **디스크 용량**: 인코딩된 HLS 파일이 원본 대비 1.5~2배 용량 차지. 디스크 모니터링 필요
- **FFmpeg 인코딩 부하**: 동시 인코딩 1~2개 제한 (CPU 부하 관리)
- **HLS 세그먼트 크기**: 6초 권장
- **서버 단일 장애점**: 서버 1대 구성이므로 장애 시 서비스 중단. 필요 시 이중화 고려

## 8. 구현 순서

### Phase 1: 프로젝트 기반 구축
- Spring Boot 4 + Gradle Kotlin DSL 프로젝트 생성
- Next.js 프로젝트 생성
- Docker Compose (MySQL)
- Nginx 설정 (리버스 프록시 + HLS 정적 파일 서빙)
- JPA 엔티티 전체 구현
- JWT 인증 시스템
- SecurityConfig

### Phase 2: 동영상 관리
- 동영상 업로드 (로컬 디스크 저장)
- FFmpeg 인코딩 파이프라인 (비동기 + 큐 관리)
- Object Storage 원본 백업 (비동기)
- 동영상 CRUD API
- 프론트엔드: 동영상 업로드, 목록, 인코딩 상태

### Phase 3: 재생목록
- 재생목록 CRUD API
- 드래그앤드롭 순서 변경
- 프론트엔드: 재생목록 에디터

### Phase 4: 시청자 페이지
- 시청자 영상 목록 / 검색
- HLS.js 비디오 플레이어
- 영상 상세 재생 페이지
- 재생목록 재생 페이지 (자동 다음 영상)
- 조회수 카운트

### Phase 5: 관리 및 마무리
- 최고관리자 패널 (사용자 관리)
- 에러 핸들링, 로깅
- Docker 컨테이너화 (전체 스택)
